# GoGoZzz - 睡觉时间记录应用

## 1. 应用概述

- **应用名称**：GoGoZzz
- **应用类型**：习惯养成类
- **核心功能**：记录睡觉时间，培养早睡习惯
- **目标用户**：希望改善睡眠习惯的成年人

## 2. 功能需求

### 2.1 打卡功能
- 只有一个打卡按钮，点击后记录当前时间
- **时间限制**：仅在 18:00 - 次日 06:00 允许打卡，其余时间禁用
- **数据保护**：已记录的时间不可修改、不可删除
- 默认正常睡觉时间：23:00
- 用户可自定义正常睡觉时间（范围：18:00 - 次日04:00）

### 2.2 主界面
- 顶部：状态栏（时间、电量等）
- 中间：巨大的圆形打卡按钮
  - 未打卡状态：显示"点击打卡"
  - 已打卡状态：显示"已入睡" + 具体时间
  - 不可打卡时段：显示灰色禁用状态 + 提示文字
- 底部：最近7天打卡记录
  - 未打卡显示**白色**
  - 已打卡按7级颜色显示

### 2.3 统计页面
- 月份选择器（支持切换月份）
- 日历热力图（当月每天的打卡情况）
- 本月数据统计：
  - 熬夜天数（7级）
  - 打卡天数
- 极值记录：
  - 本月最早睡觉时间
  - 本月最晚睡觉时间
- 趋势对比：
  - 本月 vs 上月 熬夜天数对比
  - 显示进步/退步箭头

### 2.4 分享功能
- 生成微信长图分享
- 内容包含：
  - 月度日历热力图
  - 统计数据（熬夜天数、打卡天数）
  - 趋势对比

## 3. UI/UX 设计

### 3.1 颜色方案
- 背景色：深色 (#111111, #0d0d0d)
- 打卡颜色7级（以设定23:00为例）：

| 级别 | 颜色 | 色值 | 范围 (设定23:00) |
|:---:|:---:|:---:|---|
| 1 | 🟢 深绿 | `#15803d` | < 22:20 |
| 2 | 🟢 绿色 | `#22c55e` | 22:20 ~ 22:35 |
| 3 | 🟢 浅绿 | `#4ade80` | 22:35 ~ 22:50 |
| 4 | 🟡 黄绿色 | `#a3e635` | 22:50 ~ 23:10 正常 |
| 5 | 🟡 黄色 | `#eab308` | 23:10 ~ 23:25 |
| 6 | 🟠 橙色 | `#f97316` | 23:25 ~ 23:40 |
| 7 | 🔴 红色 | `#ef4444` | > 23:40 熬夜 |

- 未打卡：白色 `#FFFFFF`

### 3.2 布局
- 底部导航：首页 + 统计 两个Tab
- 极简风格，突出核心打卡按钮

## 4. 技术方案

- **框架**：Flutter（优先Android）
- **存储**：SQLite（本地数据库）
- **分享**：图片生成库（screenshot + share_plus）

## 5. 数据库设计

### 5.1 打卡记录表 (sleep_records)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键自增 |
| date | TEXT | 日期（唯一），格式 `YYYY-MM-DD` |
| time | TEXT | 睡觉时间，格式 `HH:mm` |
| level | INTEGER | 颜色级别（1-7） |
| created_at | TEXT | 记录创建时间 |

- date 字段设唯一索引，确保每天只有一条记录
- level 根据用户设定的正常时间计算得出

### 5.2 用户设置表 (user_settings)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键（固定为1） |
| normal_time | TEXT | 正常睡觉时间，默认 `23:00` |
| updated_at | TEXT | 更新时间 |

- 只有一行配置，用户修改时更新此行

## 6. 数据流程

```
用户点击打卡
    ↓
检查时间是否在 18:00 - 次日 06:00
    ↓
计算 level：根据 normal_time 计算颜色级别
    ↓
写入 SQLite（date 唯一，不可重复打卡）
    ↓
更新首页最近7天 + 统计页面
```
